1. **ساخت مخزن در GitHub:**
   - وارد سایت [GitHub](https://github.com/) شوید و یک مخزن جدید (New Repository) بسازید (مثلاً با نام `Soheili-MDF`).

   ```powershell
   # 1. معرفی آدرس گیت‌هاب به پروژه
   git remote add origin https://github.com/YourUsername/Soheili-MDF.git

   # 2. تغییر نام شاخه اصلی به main
   git branch -M main

   # 3. فرستادن کدها به گیت‌هاب
   git push -u origin main
   ```

3. **برای دفعات بعد:**
   هر وقت تغییری در کدها دادید و خواستید در گیت‌هاب به‌روزرسانی شود، کافیست این ۳ مرحله را تکرار کنید:
   - `git add .` (آماده‌سازی تغییرات)
   - `git commit -m "توضیح کوتاه تغییر"` (ثبت تغییر)
   - `git push` (ارسال به بالا)

**Assistant**

سؤال بسیار خوبی است! این دقیقاً همان بخشی است که سایت را از حالت «ایستا» (ثابت) به «پویا» (دینامیک) تبدیل می‌کند. فرآیند جایگزینی اطلاعات در ۳ مرحله اصلی انجام می‌شود:

**۱. پیدا کردن محل تزریق در HTML**
در فایل‌های HTML، ما یک ظرف یا ظرف‌های خالی با یک شناسه خاص (ID) می‌سازیم که جاوااسکریپت بداند اطلاعات را کجا بریزد. مثلاً در [index.html](file:///c:/xampp/htdocs/Soheili-MDF/index.html#L161):
```html
<div class="products-grid" id="featuredProductsGrid">
  <!-- اینجا با جاوااسکریپت پر می‌شود -->
</div>
```

**۲. ساختن کارت‌ها با کد (تابع سازنده)**
در فایل [main.js](file:///c:/xampp/htdocs/Soheili-MDF/assets/js/main.js)، توابعی داریم که مثل یک کارخانه عمل می‌کنند. آن‌ها داده‌های خام (مثل نام محصول) را می‌گیرند و به کدهای HTML تبدیل می‌کنند.
- تابع [createProductCard](file:///c:/xampp/htdocs/Soheili-MDF/assets/js/main.js#L226-L259): این تابع با دستور `document.createElement` تگ‌های `article` و `h3` و غیره را می‌سازد و محتوا را داخل آن‌ها می‌گذارد.

**۳. تزریق به صفحه (تابع رندر)**
در نهایت، یک تابع داریم که لیست محصولات را می‌گیرد و داخل آن ظرف خالی که در مرحله اول ساختیم، می‌چسباند. به این بخش از کد [renderHomePage](file:///c:/xampp/htdocs/Soheili-MDF/assets/js/main.js#L611-L630) دقت کن:

```javascript
function renderHomePage() {
  // ۱. پیدا کردن ظرف خالی در صفحه
  var productsGrid = document.getElementById("featuredProductsGrid");

  if (productsGrid) {
    productsGrid.innerHTML = ""; // خالی کردن محتوای قبلی
    
    // ۲. بریدن ۶ تا از محصولات و چرخیدن روی تک‌تک آن‌ها
    products.slice(0, 6).forEach(function (p) {
      // ۳. ساخت کارت برای هر محصول و اضافه کردن به ظرف (appendChild)
      productsGrid.appendChild(createProductCard(p));
    });
  }
}
```

**خلاصه دستورات کلیدی:**
- `document.getElementById("ID")`: برای پیدا کردن محل نمایش در صفحه.
- `document.createElement("tag")`: برای ساختن یک تگ جدید.
- `element.textContent = data`: برای ریختن متن (مثلاً نام محصول) داخل تگ.
- `parent.appendChild(child)`: برای چسباندن تگ ساخته شده به داخل صفحه.

با این روش، شما فقط کافیست لیست محصولات را در ابتدای [main.js](file:///c:/xampp/htdocs/Soheili-MDF/assets/js/main.js) تغییر دهید؛ بقیه کارها (ساخت کارت و نمایش در صفحات مختلف) به صورت خودکار توسط این کدها انجام می‌شود.

**تغییراتی که برای بهبود نمایش تصاویر انجام دادم:**

- **افزایش ارتفاع:** ارتفاع باکس تصاویر در کارت‌های محصول و پروژه را بیشتر کردم (`220px` و `240px`) تا جزئیات تصویر بهتر دیده شود.
- **افکت زوم حرفه‌ای:** قابلیتی اضافه کردم که وقتی ماوس را روی کارت می‌برید، تصویر به نرمی کمی بزرگ می‌شود (Zoom-in). این کار حس پویایی و کیفیت بالایی به سایت می‌دهد.
- **اصلاح چیدمان (Object-fit):** تنظیمات را طوری تغییر دادم که تصاویر کل فضای باکس را پر کنند بدون اینکه کشیده یا دفرمه شوند (`object-fit: cover`).
- **حذف پس‌زمینه‌های مزاحم:** گرادینت‌های رنگی که قبلاً برای حالت بدون تصویر گذاشته بودم را برداشتم تا با رنگ تصاویر واقعی شما تداخل نداشته باشند.

**Assistant**

این ۴ دستور (و ترتیبشون) یعنی «تغییراتم رو ببین → آماده ارسال کن → ثبتش کن → بفرست روی گیت‌هاب». مفهوم هر کدوم:

## 1) `git status`
- کارش: وضعیت الان پروژه رو بهت می‌گه.
- به زبان ساده: می‌گه کدوم فایل‌ها تغییر کردن، کدوم‌ها آماده ثبت شدن (staged)، و کدوم‌ها هنوز ثبت/ارسال نشدن.
- خروجی معمولاً ۳ حالت داره:
  - **Untracked**: فایل جدیدی که گیت هنوز نمی‌شناسد.
  - **Modified**: فایل تغییر کرده.
  - **Staged**: فایل تغییر کرده و آماده‌ی commit است.

## 2) `git add .`
- کارش: همه تغییرات فعلی پوشه پروژه رو می‌بره به مرحله **Stage** (آماده ثبت).
- به زبان ساده: داری می‌گی «این تغییرات رو می‌خوام توی ثبت بعدی (commit) حساب کنی».
- نکته: نقطه `.` یعنی «همه چیز در همین پوشه و زیرپوشه‌ها».

اگر نخواستی همه چیز رو اضافه کنی:
```bash
git add index.html
```

## 3) `git commit -m "update"`
- کارش: از چیزهایی که stage کردی یک «عکس/نسخه» در تاریخچه گیت ثبت می‌کند.
- به زبان ساده: مثل Save کردن یک نسخه رسمی از پروژه با یک پیام توضیح.
- `-m` یعنی پیام کامیت را همانجا می‌نویسی.
- نکته مهم: فقط چیزهایی که **stage** شده‌اند وارد commit می‌شوند (یعنی بعد از `git add`).

مثال پیام بهتر:
```bash
git commit -m "Add project detail page and pagination"
```

## 4) `git push`
- کارش: commitهایی که روی سیستم خودت ساخته‌ای را می‌فرستد به مخزن آنلاین (GitHub).
- به زبان ساده: «آپلود کردن» تغییرات روی گیت‌هاب.
- نکته: اگر اولین بار push به یک branch جدید باشد معمولاً این را می‌زنند:
```bash
git push -u origin main
```
ولی بعد از یکبار، همان `git push` کافی است.

`origin` و `main` دو تا مفهوم جدا هستن که معمولاً کنار هم توی دستور `git push` می‌بینیشون:

## origin یعنی چی؟
- **origin** اسمِ پیش‌فرضِ «ریموت» (Remote) است.
- ریموت یعنی **آدرس مخزن آنلاین** شما (مثلاً روی GitHub).
- وقتی می‌گی:
  ```bash
  git push origin main
  ```
  یعنی «تغییرات را بفرست به مخزن آنلاین‌ای که اسمش origin است».

می‌تونی ریموت‌ها رو ببینی:
```bash
git remote -v
```

## main یعنی چی؟
- **main** اسمِ «برنچ» (Branch) اصلی پروژه است.
- برنچ یعنی یک شاخه از تاریخچه‌ی کدها (مثل مسیر اصلی توسعه).
- وقتی می‌گی:
  ```bash
  git push origin main
  ```
  یعنی «برنچ main روی کامپیوترم رو بفرست روی origin».

## پس `git push origin main` دقیقاً یعنی:
«برنچ mainِ پروژه‌ی من را در ریموت origin (GitHub) آپلود/به‌روز کن.»

## `-u` توی `git push -u origin main` چیه؟
- `-u` یعنی از این به بعد، گیت یادش باشد که `main` محلی به `origin/main` وصل است.
- بعد از یکبار استفاده از `-u`، دفعه‌های بعد فقط این کافی است:
  ```bash
  git push
  ```

اگر خواستی، اسم origin هم می‌تواند هرچیز دیگری باشد (مثلاً `github`) ولی معمولاً همه همان `origin` می‌گذارند.
